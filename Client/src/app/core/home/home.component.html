So how it works?

This section explains the purchase process from adding to the shopping cart to completing an order.

  1. In the shop, when changing the shopping cart by decreasing/increasing the quantity or removing/adding items, the cart is getting saved with a debounce time, following this approach saves HTTP requests every time the cart is changed.
  2. Checkout, at this point, the user has to be logged in, you can easily logged in using the Demo-login or going with full authentication of registration, once you are logged in, the checkout route is accessible.
  3. In the checkout, there are 4 steps to complete.
    • Address
    • Delivery method
    • Payment
    • Order review
    Once the checkout is completed you will be navigated to the complete page with the placed order summary.
  4. For the payment part, the app uses Strong Customer Authentication(SCA) process, at first, payment intent is created at the Order review step at checkout, and the client secret is stored in the client app.
  5. At this point, the user can quit the checkout and make any changes to his shopping cart and the whole process repeat itself.
  6. Once the "Place Order" button is hit, the order gets created at the backend and here two cases can occur:
    • Success case, the order is created with "pending" status and once payment is received the order should get updated to "Received" status using stripe Webhooks. (Note: Realtime or WebSockets are not implemented here, you will have to refesh the page to see updated status since the app also uses Cache mechanics)
    • Failed case - any reason that could be caused, an invalid payment, stolen card, or no-money case, in these cases the order still gets created and the user can re-try to make the payment and the created order gets updated. (status, items, price.. etc)
  7. Next, you can view your orders in the "orders" tab and order details for each with more information.
  8. End of my summary.
